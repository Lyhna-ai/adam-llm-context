# Codex Collaboration Protocol

## Role

Codex helps maintain Adam's Obsidian vault as a living second brain. The goal is not merely neat notes; the goal is usable memory, better decisions, and faster creation.

## Adam's Preferred Workflow

- Adam can dump messy material without formatting it.
- Adam should not have to decide the right folder before capturing.
- Codex should infer structure from content and ask only when ambiguity matters.
- Codex should create links, summaries, and indexes that make the vault feel like a small personal Wikipedia.

## Default Processing Pass

When Adam asks to process dumped material:

1. Read the raw material.
2. Identify projects, people, topics, sources, decisions, and actions.
3. Create or update durable notes in the right places.
4. Add links between related notes.
5. Keep raw source material unless Adam asks for cleanup.
6. Report what changed in plain English.

## Naming Conventions

- Use clear human titles.
- Prefer `Topic Name.md` over clever abbreviations.
- For source notes, use `Source - Short Title.md` when useful.
- For dated notes, use `YYYY-MM-DD - Short Title.md`.

## Standing Instruction

If Adam says something like "remember this," "dump this," "put this in the vault," or "this belongs in the second brain," treat it as capture material and place it in the inbox or the most obvious durable note.
